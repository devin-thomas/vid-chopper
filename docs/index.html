Qt 6 Desktop Utility

VidChopper

VidChopper turns a single source video into chapter clips with a focused Windows desktop workflow, precise timestamp editing, GPU-aware encoding defaults, and a testable native C++ core.

View Repository Build Notes

What It Does

Load a video, start from embedded chapters or a six-part starter layout, edit names and boundaries, and export each chapter through ffmpeg.

Why It Exists

The project was built as a practical personal tool and a portfolio-grade desktop application: native code, explicit tradeoffs, and a repository meant to be reviewed, discussed, and reused.

Encoding Defaults

VidChopper prefers HEVC NVENC when both an NVIDIA GPU and hevc_nvenc are available. Otherwise it falls back to x264 for predictable compatibility.

Timestamp Modes

Edit boundaries in milliseconds or in frame notation. Frame-mode inputs use the source video’s probed frame rate and are meant for tight chapter cuts without mental conversion.

Repository Highlights

  • Qt 6 Widgets interface with a dark-only workflow
  • Pure C++ core for chapter planning and command generation
  • Fast unit tests plus slower ffmpeg-backed integration coverage
  • GitHub Actions CI for both the core and GUI build paths

Important Constraints

  • Windows is the primary target in the current repository state
  • Full GUI builds require a Qt 6 SDK
  • Exports are sequential to keep validation and progress tracking clear
  • Qt and FFmpeg licensing obligations remain separate from the repo license
