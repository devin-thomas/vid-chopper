# Pages Rewrite Brief

This was the Task 10 brief and now serves as the reference for future iteration.

## Site Role

The site had to become multi-page and cover all three of these roles at once:

- product landing page
- release portal
- developer docs site

## Design Direction

- Theme the site around the VidChopper icon.
- Use Thumio as the design bible.
- Keep the visual system clean rather than noisy or generic.
- Lean cinematic and creator-tool adjacent without turning into an overdone media-brand landing page.

## Homepage Priorities

1. Download the release ZIP
2. Show screenshots and workflow previews that funnel toward download
3. Explain the feature value quickly enough to support the download decision

Explicit copy constraint:

- do not directly encourage contributions on the homepage

## Shipped IA

- `/`
  - product landing page
  - primary download CTA
  - screenshot-led workflow persuasion
- `/download`
  - release portal
  - current version, package facts, install notes, and release history link
- `/features`
  - feature and workflow breakdown
- `/docs`
  - developer-oriented docs entry point and source-doc links
